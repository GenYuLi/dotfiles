# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Is

Nix flake-based dotfiles managing neovim, zsh, tmux, and desktop environment across multiple platforms (standalone home-manager, NixOS, nix-darwin). Catppuccin Macchiato theme throughout.

## Build Commands

```bash
# Bootstrap (first time or changing profile)
./setup.py --bootstrap                          # interactive — generates config/default.nix
./setup.py --bootstrap --profile home           # explicit profile: home | darwin | nixos

# Apply configuration (switch)
./setup.py                                      # build and switch to current profile
./setup.py --build                              # build only, no switch
./setup.py --dry                                # dry run — prints the command without executing

# Quick rebuild alias (defined in zsh.nix)
dotswitch                                       # home-manager switch / nixos-rebuild switch / darwin-rebuild switch

# Pre-commit checks (run from devShell)
nix develop                                     # enters devShell with pre-commit hooks installed
nix flake check                                 # runs pre-commit checks: editorconfig-checker, nixpkgs-fmt, stylua
```

## Architecture

### Flake Structure

- **`flake.nix`** — Entry point. Defines inputs (nixpkgs 25.11 + unstable), outputs for home/nixos/darwin configs, pre-commit checks, and devShell.
- **`lib/default.nix`** — Core helpers: `mkHome` (home-manager config), `mkSystem` (NixOS/Darwin config), pkgs instantiation with overlays. Reads `config/default.nix` for system metadata.
- **`config/default.nix`** — Generated by `setup.py --bootstrap`. Contains system, profile, username, hostname, email, signingKey. **Do not edit manually.**
- **`setup.py`** — Python bootstrap/build script. Generates `config/default.nix`, then delegates to `nix run` with the appropriate tool (home-manager/nixos-rebuild/darwin-rebuild).

### Module Layout

| Directory | Purpose |
|-----------|---------|
| `home/` | Home-manager modules: `default.nix` (packages + programs), `zsh.nix`, `tmux.nix`, `fzf.nix`, `skim.nix`, `git.nix`, `alacritty.nix`, `ghostty.nix`, `desktop.nix`, `cpp.nix` |
| `config/` | App configs symlinked to XDG dirs: `nvim/` (Lua, lazy.nvim), `zsh/`, `tmux/`, `navi/`, `niri/` (KDL), `waybar/`, etc. |
| `system/common/` | Shared system config (home-manager integration) |
| `system/nixos/` | NixOS-specific: hardware, gaming, idle, logiops |
| `system/darwin/` | macOS-specific: homebrew, system defaults |
| `lib/nix.nix` | Nix daemon settings shared across profiles |
| `nix/shell/` | Reusable devShell templates (FHS, venv, pyproject) |
| `docs/` | Chinese-language reference docs |

### Neovim Config (`config/nvim/`)

Lua-based with lazy.nvim. Core in `lua/core/` (options, mappings, autocmds, bootstrap). Plugins in `lua/plugins/` (~30 configs). LSP support for Go, Rust, Python, C++, Nix, JSON, and more. Uses 2-space indentation.

**LSP architecture notes** (Neovim 0.12 + native LSP API):

- LSP servers are enabled by dropping a `<server>.lua` config file into `config/nvim/after/lsp/` — `lua/plugins/lsp/init.lua` iterates that directory and calls `vim.lsp.enable()` for each.
- **Rust LSP is managed by `rustaceanvim` (`lua/plugins/rustaceanvim.lua`), NOT lspconfig.** Do not add `after/lsp/rust_analyzer.lua` — that would spawn a second `rust_analyzer` client alongside rustaceanvim's `rust-analyzer` client, causing duplicate `gd` results, doubled diagnostics, etc. mason-lspconfig is configured with `automatic_enable = { exclude = { "rust_analyzer" } }` to enforce this. rust-analyzer settings live in `vim.g.rustaceanvim.server.default_settings['rust-analyzer']`, NOT in an `after/lsp/` file.
- **rust-analyzer config gotcha**: `checkOnSave` is a top-level boolean (sibling to `cargo`, `check`, `procMacro`). Nesting it as `check.onSave` is silently ignored — that key doesn't exist in rust-analyzer's schema.
- `:LspInfo` and `:LspRestart` are user-command **shims** defined in `lua/plugins/lsp/init.lua`. Newer nvim-lspconfig removed the originals; modern equivalents are `:checkhealth vim.lsp` and `vim.lsp.stop_client(...)`. The shims exist for finger memory.
- `:RustLsp` is **buffer-local** (rustaceanvim creates it in `on_attach`). Only available inside `.rs` buffers where rust-analyzer has attached; not visible elsewhere or in global `:command` listing.

### Key Patterns

- **Nixpkgs channels**: Stable (`nixos-25.11`) with unstable overlay for select packages (neovim, tmux).
- **Unstable packages**: `pkgs.unstable.<pkg>` overlay defined in `lib/default.nix` gives access to nixpkgs-unstable.
- **Profile types**: `home` (home-manager only), `darwin` (macOS + home-manager), `nixos` (NixOS + home-manager).
- **Platform conditionals**: `pkgs.stdenv.isDarwin` / `pkgs.stdenv.isLinux` and `isSystemConfig` (true for NixOS/Darwin, false for standalone home-manager).
- **XDG symlinks**: App configs in `config/` are symlinked via `xdg.configFile.<name>.source` in home-manager modules.
- **State version**: Derived automatically from flake.lock's home-manager ref (`lib/default.nix`), currently 25.11.

## Formatting & Style

- **Nix**: Format with `nixpkgs-fmt`. Pre-commit hook enforces this.
- **Lua** (neovim config): Format with `stylua` using `config/nvim/stylua.toml`.
- **EditorConfig**: 2-space indent, UTF-8, LF line endings, trim trailing whitespace.
- New files must be `git add`ed before `nix flake` can see them (flake purity).

## Files to Never Commit

- **`config/default.nix`** — Auto-generated by `setup.py --bootstrap` with machine-specific values. Never commit filled-in values; only commit structural changes (e.g., adding new fields with empty defaults).
- **`config/nvim/lazy-lock.json`** — Only commit when explicitly asked. Do not auto-stage; plugin pin updates from lazy.nvim are noise unless intentional. When the file does need to be committed (e.g., bringing in upstream pin updates from a merge), strip entries for `overlook.nvim` and `profile.nvim` so those two plugins remain unpinned. Pin updates for other plugins are fine.
